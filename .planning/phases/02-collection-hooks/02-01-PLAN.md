  <action>
Create `src/hooks/shared.ts` with two exports:

1. `readStdin()`: Returns Promise<string>. Reads all stdin into a buffer using process.stdin events ('data', 'end', 'error'). Sets encoding to 'utf-8'. Resolves on 'end', rejects on 'error'.

2. `summarizeToolInput(toolName: string, toolInput: Record<string, unknown>): string`: Produces a concise summary capped at 200 characters. Switch on toolName:
   - 'Bash': return truncate(String(toolInput.command ?? ''), 200)
   - 'Write', 'Edit', 'Read': return truncate(String(toolInput.file_path ?? ''), 200)
   - 'Glob': return truncate(String(toolInput.pattern ?? ''), 200)
   - 'Grep': return truncate(String(toolInput.pattern ?? ''), 200)
   - default: return truncate(JSON.stringify(toolInput), 200)

Helper function `truncate(str: string, maxLen: number): string` returns str.slice(0, maxLen) + '...' if str.length > maxLen, else str.

Append tests for summarizeToolInput and readStdin to `tests/unit/hooks/shared.test.ts`. For readStdin, use a mock approach: create a Readable stream, pipe it, test the output. Vitest's vi.spyOn can mock process.stdin if needed, or use a helper that accepts a Readable.

Note: For testability, also export a version `readFromStream(stream: NodeJS.ReadableStream): Promise<string>` that readStdin delegates to. Tests call readFromStream with a mock Readable.
