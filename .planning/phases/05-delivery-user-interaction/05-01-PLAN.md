<objective>
Build the core delivery infrastructure: Zod schemas for recommendation state, config extensions for delivery settings, markdown renderer for AnalysisResult, state tracking for recommendation lifecycle, and rotator for keeping the recommendations file bounded.

Purpose: This plan establishes all the data contracts and core functions that Plans 02 and 03 depend on. Without the renderer, no recommendations can be written to the .md file. Without state tracking, recommendations cannot be applied/dismissed. Without the rotator, the file grows unbounded.

Output: Six new source files (schemas, renderer, state, rotator, delivery index), config extension, paths extension, and three test files.
</objective>

    **Step 4: Create src/delivery/renderer.ts**

    Pure function `renderRecommendations(result: AnalysisResult, states: Map<string, RecommendationStatus>): string`.

    - Header: `# harness-evolve Recommendations`
    - Metadata: `*Generated: {generated_at}*` and `*Period: {since} to {until} ({days} days)*`
    - Group recommendations by confidence tier in order: HIGH, MEDIUM, LOW
    - For each tier with recommendations: `## {TIER} Confidence`
    - For each recommendation:
      - `### [{STATUS}] {title}` where STATUS is from states map, defaulting to "PENDING"
      - `**Target:** {target} | **Pattern:** {pattern_type}`
      - `**Evidence:** {count} occurrences across {sessions} sessions` (omit sessions if undefined)
      - Empty line, then `description`
      - `**Suggested action:** {suggested_action}`
      - If `ecosystem_context` is not undefined: `**Ecosystem note:** {ecosystem_context}`
    - If no recommendations at all: output header + `\nNo recommendations at this time.\n`
    - Footer: `---\n*Run /evolve to refresh or manage recommendations.*`

    **Step 5: Create src/delivery/state.ts**

    Functions for recommendation state lifecycle:

    - `loadState(): Promise<RecommendationState>` -- reads from `paths.recommendationState`. If file doesn't exist, returns `{ entries: [], last_updated: new Date().toISOString() }`. Validates with `recommendationStateSchema.parse()`.
    - `saveState(state: RecommendationState): Promise<void>` -- writes atomically using `write-file-atomic` to `paths.recommendationState`.
    - `updateStatus(id: string, status: RecommendationStatus, details?: string): Promise<void>` -- loads state, finds entry by id (or creates new), updates status and `updated_at`, saves. If status is "applied", sets `applied_details` to details param.
    - `getStatusMap(): Promise<Map<string, RecommendationStatus>>` -- loads state, returns `new Map(state.entries.map(e => [e.id, e.status]))`.

    **Step 6: Create src/delivery/rotator.ts**

    Function `rotateRecommendations(config: { maxRecommendationsInFile: number; archiveAfterDays: number }): Promise<void>`:

    1. Load state via `loadState()`
    2. Calculate cutoff date: `new Date(Date.now() - config.archiveAfterDays * 86400_000)`
    3. Split entries into:
       - `toArchive`: entries where status is "applied" or "dismissed" AND `updated_at` < cutoff
       - `toKeep`: everything else
    4. If `toArchive.length === 0`, return early (nothing to do)
    5. Write archive: `paths.recommendationArchive + '/' + YYYY-MM-DD.json` (today's date) using `write-file-atomic`. Content is JSON array of archived entries.
    6. Save state with only `toKeep` entries.
