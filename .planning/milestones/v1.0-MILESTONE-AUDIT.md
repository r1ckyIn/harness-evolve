### inferPatternType Mismatch Detail

`src/analysis/outcome-tracker.ts` `inferPatternType()` returns different strings than classifiers' actual `pattern_type` values for 7/8 classifiers:

| Classifier | Produces | outcome-tracker infers | Impact |
|-----------|----------|----------------------|--------|
| repeated-prompts | `repeated_prompt` | `repeated-prompt` | adjustConfidence miss |
| long-prompts | `long_prompt` | `long-prompt-workflow` | adjustConfidence miss |
| code-corrections | `code_correction` | `code-correction-pattern` | adjustConfidence miss |
| personal-info | `personal_info` | `unknown` (fallthrough) | adjustConfidence miss |
| config-drift | `config_drift` | `unknown` (fallthrough) | adjustConfidence miss |
| ecosystem-adapter | varies | `ecosystem-adapter` | adjustConfidence miss |
| onboarding | varies | `onboarding` | adjustConfidence miss |
| **permission-patterns** | **`permission-always-approved`** | **`permission-always-approved`** | **MATCH** |

**Impact:** `adjustConfidence` in analyzer.ts only works for permission-always-approved pattern type. All other pattern types' outcome histories are tracked but don't influence future confidence tiers. Core pipeline unaffected.

**Suggested fix for v2:** Align `inferPatternType` return values to match classifier `pattern_type` values.
